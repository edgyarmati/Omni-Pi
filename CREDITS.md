# Credits

Omni-Pi exists because of the Pi ecosystem and the work of earlier authors.

## Core upstream foundations

- [badlogic/pi-mono](https://github.com/badlogic/pi-mono) by Mario Zechner and contributors
  - provides the Pi runtime, package model, extension system, skill loading, and terminal agent foundation Omni-Pi builds on

## Workflow and orchestration inspiration

- [can1357/oh-my-pi](https://github.com/can1357/oh-my-pi)
  - inspiration for model-role routing, orchestration ideas, and Pi-native workflow enhancements
- [gsd-build/gsd-2](https://github.com/gsd-build/gsd-2)
  - inspiration for disk-first workflow state, durable progress files, and a batteries-included Pi-based product shape

## Ecosystem inspiration

- [nicobailon/pi-subagents](https://github.com/nicobailon/pi-subagents) by Nico Bailon
  - used in Omni-Pi as the isolated worker/expert execution substrate
- nicopreme / nicobailon Pi ecosystem work more broadly
  - inspiration for subagent and extension-driven workflows
- the broader Pi community
  - package, skill, and workflow ideas that shaped the Omni-Pi direction

## Attribution intent

Omni-Pi is an original project, but it is intentionally built in conversation with upstream Pi work.
This repository should continue to keep visible credit in `README.md` and `CREDITS.md` as the project evolves.
